## Codebase Patterns
- This repository is Python-first; use `python -m mypy docos/` for type checking and `python -m pytest tests/ -q` for automated verification. Use `pip install -i https://pypi.tuna.tsinghua.edu.cn/simple` for network-reliable installs.
- Batch verification entrypoints live in `scripts/quick_verify_papers.py` and `scripts/download_and_verify_papers.py`.
- Pipeline quality signals already flow through `docos/pipeline/runner.py`, `docos/run_store.py`, `docos/artifact_stores.py`, `docos/review/queue.py`, and `docos/wiki/compiler.py`; prefer reusing those artifacts instead of inventing parallel state.
- Focus test coverage on small targeted files under `tests/` and reuse fixture helpers from `tests/fixtures/build_fixtures.py` when batch or parser fixtures are needed.
- `PipelineResult.gate_passed` is `bool | None` (None = gate stage never reached); `RunManifest.review_status` can be `None`, `"pending"`, `"auto_merged"`, `"approved"`, `"rejected"`, `"changes_requested"`.
- Verdict tier classification lives in `_classify_verdict()` in `scripts/quick_verify_papers.py`; downstream stories should reuse this function rather than re-implementing gate/review logic.
