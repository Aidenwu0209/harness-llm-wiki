## Codebase Patterns
- This repository is Python-first; use `python -m mypy docos/` for type checking and `python -m pytest tests/ -q` for automated verification.
- The main runtime boundaries are `docos/cli`, `docos/pipeline`, `docos/wiki`, `docos/lint`, `docos/harness`, and `docos/review`; keep stories scoped to one boundary when possible.
- Test files follow naming pattern `tests/test_<feature>_usXXX.py`
- US-001 through US-036 features may have been implemented in a previous PRD cycle — always check existing code and tests first
- Config YAML for tests: use inline strings with `environment: local\nschema_version: '1'` header
- Registered parsers: `stdlib_pdf` and `basic_text_fallback` (NOT "basic_text")
- PIPELINE_STAGES: ingest, route, parse, normalize, extract, compile, patch, lint, harness, gate
