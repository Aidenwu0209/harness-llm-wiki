## Codebase Patterns
- This repository is Python-first; use `python -m mypy docos/` for type checking and `python -m pytest tests/ -q` for automated verification.
- The main runtime boundaries are `docos/pipeline`, `docos/wiki`, `docos/review`, `docos/lint`, `docos/harness`, and `docos/cli`; keep each story focused on one boundary when possible.
- Tests use inline YAML config strings with `stdlib_pdf` as primary_parser; config_dir pattern via `_make_test_config()`.
- Fixtures are built on-demand via `tests/fixtures/build_fixtures.py` using `get_fixture(name)` or `get_all_fixtures()`.
